# Lessons Learned - Team 11 Store

## Technical Skills Learned
- How to build a REST API using Express.js
- How to connect Node.js to MongoDB Atlas using Mongoose
- How to perform CRUD operations (Create, Read, Update, Delete)
- How to structure a backend project using controllers, models, and routes
- How to test APIs using Jest and Supertest
- How to use Thunder Client to manually test endpoints

## Challenges Faced
- Setting up MongoDB Atlas connection
- Fixing PowerShell execution policy issues when running npm commands
- Understanding how routes, controllers, and models connect together
- Handling async operations and errors properly

## Solutions
- Used environment variables (.env) to manage configuration
- Used middleware and proper routing structure
- Debugged step by step using console logs and test tools
- Followed a modular structure for better organization

## What We Would Improve
- Add a frontend UI for better user interaction
- Add authentication and user accounts
- Improve error handling and validation
- Deploy the application online (e.g., Render or Vercel)

## Team Contribution
All team members contributed to the design, development, and testing of the application. Collaboration helped in understanding different parts of the system and completing the project successfully.
